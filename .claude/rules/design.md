# 前端设计规范

## 优先级

设计规范的优先级从高到低：

1. **`preview/` 目录** — 设计的唯一权威来源（如果存在）。其中 `preview/design-direction.md` 由 `/taste` 探索产出，是项目设计方向的权威落点
2. **项目 spec.md 中的设计方向** — 项目级定制
3. **本文件** — 通用默认规范

当 `preview/` 或 spec.md 中的设计方向与本文件的通用规则冲突时，**以项目方向为准**。例如：本文件说"禁止蓝色主按钮"，但项目方向定义了蓝色为主色，则遵循项目方向。本文件的规则仅在没有明确设计方向时生效。

> 不知道项目该用什么风格时，用 `/taste` 探索定向（详见 `commands/taste.md`）——它产出 `preview/design-direction.md`，即上面优先级链里的第 1 项。

## 必须做的事

### 视觉层次
- 每个页面必须有清晰的信息层次：标题 > 正文 > 辅助文字
- 通过字号（而非颜色）区分层次：标题 18-24px，正文 14-16px，辅助 11-13px
- 最重要的信息放在最显眼的位置，不要让用户找

### 留白
- 卡片之间至少 12px 间距
- 卡片内部至少 14px 内边距
- 页面左右边距至少 16px（移动端）
- 宁可多留白，不要挤在一起。留白 = 高级感，拥挤 = 廉价感

### 配色
- 遵循 60-30-10 法则：60% 背景色、30% 辅助色、10% 强调色
- 主按钮只能用一种颜色，全站统一
- 图标背景可以用彩色，但要低饱和度
- 图表用柔和色系，不用高饱和色
- 默认禁止：紫色渐变白底、彩虹色齐上阵、高饱和度大面积使用

### 圆角与阴影
- 同一类元素圆角必须一致（如所有卡片统一 16px，所有按钮统一 12px）
- 不同类型的元素可以用不同圆角（卡片 16px、按钮 12px、标签 20px）
- 阴影要极淡，`rgba(0,0,0,0.04)` 级别，不要明显的投影
- 不要同时使用边框和阴影（二选一）

### 字体
- 中文项目优先：`-apple-system, 'PingFang SC', 'Hiragino Sans GB', sans-serif`
- 英文项目：拒绝 Arial/Roboto/Inter，选有个性的字体
- 全站字体不超过 2 种（一个标题用，一个正文用）

### 交互
- 可点击元素必须有 hover/press 反馈（颜色变化或轻微缩放）
- 输入框焦点态必须有视觉变化（边框色或阴影）
- 动效时长 0.15-0.3s，使用 ease 过渡
- 不要使用弹跳、闪烁等花哨动效

## 默认禁止

以下规则在没有明确风格方向时生效。当 `preview/`（含 `/taste` 产出的 design-direction.md）或 spec.md 中的设计方向与这些规则冲突时，以项目方向为准。

1. 紫色渐变白底背景（AI 廉价感标配）
2. 彩虹色齐上阵
3. 高饱和度大面积使用
4. 所有元素用同一个圆角值（卡片、按钮、标签应有区分）
5. 没有留白的拥挤布局
6. 无意义的装饰动画
7. 使用 emoji 代替图标
8. 纯黑 `#000000` 做背景（用 `#0a0a0a` 或 `#111`）

## 项目级设计方向

每个项目在 spec.md 中必须声明设计方向，Builder 必须遵循。例如：

```
## 设计方向
- 风格：Apple 风格白底简约
- 主色调：黑白灰，克制使用彩色
- 目标用户：中国年轻消费者（18-35 岁）
- 参考：preview/ 目录下的原型文件
```

如果项目没有声明设计方向，Builder 实现 UI 前必须先问 Architect-Planner。

## 设计自检清单

Builder 完成 UI 代码后，自行检查：
- [ ] 信息层次是否清晰（标题/正文/辅助文字有区分）
- [ ] 留白是否充足（不拥挤）
- [ ] 颜色是否遵循 60-30-10 法则
- [ ] 圆角和阴影是否一致
- [ ] 交互反馈是否到位（hover/focus/active）
- [ ] 移动端是否可用（触控区域 ≥ 36px）
