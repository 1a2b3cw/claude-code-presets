# 项目规划技能

## 触发条件
- 用户提出新功能需求
- 用户要求规划项目
- 开始新的开发任务

## 规划流程

### 第一步：需求澄清
向用户确认：
1. 目标用户是谁？
2. 核心问题是什么？
3. 成功标准是什么？
4. 有什么约束？（时间、技术、资源）
5. 有参考产品吗？

### 第二步：任务分解（WBS）
```
Epic（大功能）
├── Story（用户故事）
│   ├── Task（具体任务，1-2小时）
│   └── Task
└── Story
    └── Task
```

### 第三步：优先级排序
- **P0**：核心功能，没有它产品不能用
- **P1**：重要功能，显著提升体验
- **P2**：锦上添花，可以延后

### 第四步：风险评估
对每个风险评估：
- 发生概率（高/中/低）
- 影响程度（高/中/低）
- 缓解方案

### 第五步：输出规划文档
按照 Architect-Planner Agent 的 spec.md 和 tasks.md 格式输出。

## 注意事项
- 任务粒度适中：太大无法执行，太小浪费时间
- 考虑依赖关系：哪些任务可以并行，哪些必须串行
- 预留缓冲时间：意外情况总是会发生
- 优先 MVP：先做最小可用版本，再迭代
